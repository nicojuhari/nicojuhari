# Clean Code & Refactoring

Improve existing code by simplifying, restructuring, and optimizing it.

---

## When to Apply

- Reviewing code  
- Refactoring features  
- Improving readability  
- Fixing messy or complex logic  

---

## Core Principles

- Simplicity over cleverness  
- Readability over brevity  
- Remove duplication  
- Keep logic easy to follow  

---

## Rules

### 1. Identify Problems

- Large components or functions  
- Duplicated logic  
- Poor naming  
- Unclear structure  

---

### 2. Refactor

- Split large functions/components  
- Extract reusable logic  
- Rename variables for clarity  
- Remove unnecessary complexity  

---

### 3. Optimize

- Reduce re-renders  
- Simplify conditions  
- Remove dead code  

---

### 4. Preserve Behavior

- Do not break existing functionality  
- Keep logic consistent  

---

## Output Format

1. Issues found  
2. Improved code  
3. Short explanation
