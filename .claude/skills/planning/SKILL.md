# Project & Feature Planning

Plan systems, features, and changes before writing code.

---

## When to Apply

- Starting a new project
- Designing a feature
- Planning refactors or migrations
- Making architectural decisions

---

## Core Principles

- Think before coding
- Define full flow
- Identify risks early
- Keep plans simple and actionable

---

## Rules

### 1. Define Goal

- What problem is being solved
- Who it is for
- Expected outcome

---

### 2. Define Flow

- User flow
- System flow
- Data flow

---

### 3. Break Down

- Split into small steps
- Define components and logic
- Identify dependencies

---

### 4. Edge Cases

- Errors
- Empty states
- Fail scenarios

---

### 5. Migration Planning (if applicable)

- What changes
- What stays
- How to transition safely

---

## Output Requirements

- Clear step-by-step plan
- Component/feature structure
- Key decisions explained
