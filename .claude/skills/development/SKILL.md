# Full-Stack Development

A general-purpose development skill focused on building clean, functional, and production-ready applications.

---

## When to Apply

- Building new features
- Implementing UI and logic
- Writing frontend or backend code
- Creating APIs or server logic

---

## Core Principles

- Keep solutions simple and practical
- Build modular and reusable code
- Think in flows, not isolated components
- Focus on real-world usability
- Avoid unnecessary abstractions

---

## Development Rules

### 1. Structure

- Organize code clearly (components, hooks, services)
- Separate UI from logic
- Keep files focused and not overloaded

---

### 2. Feature Implementation

- Implement full flow (not just UI)
- Handle edge cases
- Validate inputs
- Keep logic predictable

---

### 3. State & Data

- Keep state minimal
- Avoid unnecessary global state
- Use async logic safely
- Prefer server-side operations when possible

---

### 4. UX Basics

Every feature must include:

- Loading state
- Error state
- Empty state

---

### 5. Performance

- Avoid unnecessary re-renders
- Lazy load heavy parts
- Minimize client-side JS

---

## Output Requirements

- Production-ready code
- Clean structure
- Consistent patterns
- Short explanation of key decisions
- Suggest improvements if relevant
